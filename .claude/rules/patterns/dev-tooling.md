# Dev Tooling

## Laravel Pint (PHP)

Pint formats all PHP files using the Laravel preset.

```bash
vendor/bin/pint --dirty --format agent    # Format changed files (use after modifying PHP)
vendor/bin/pint --format agent            # Format all files
```

**Always run `vendor/bin/pint --dirty --format agent` after modifying PHP files.**

## ESLint (TypeScript + Svelte)

Flat config in `eslint.config.js`. Lints `.ts`, `.svelte`, `.svelte.ts` files.

**Key rules:**
- `@typescript-eslint/consistent-type-imports`: enforce separate type imports
- `import/order`: alphabetical import sorting by group
- `@stylistic/brace-style`: 1TBS, no single-line blocks
- `@stylistic/padding-line-between-statements`: blank lines around control flow
- `curly`: always require braces

**Ignored paths:** `vendor`, `node_modules`, `public`, `bootstrap/ssr`, `resources/js/actions/**`, `resources/js/components/ui/*`, `resources/js/routes/**`, `resources/js/wayfinder/**`

```bash
npx eslint .                    # Check
npx eslint --fix .              # Auto-fix
```

## Prettier (Svelte)

Formats `.svelte` files. ESLint handles `.ts` files.

**Plugins:** `prettier-plugin-svelte`, `prettier-plugin-tailwindcss`

```bash
npx prettier --check '**/*.svelte'
npx prettier --write '**/*.svelte'
```

## package.json Scripts

```bash
npm run build          # Production build
npm run dev            # Dev server with HMR
```

## Artisan Commands

```bash
php artisan serve                    # Start PHP dev server
php artisan test --compact           # Run tests
php artisan migrate                  # Run migrations
php artisan make:model Name -mf     # Model + migration + factory
php artisan make:controller Name     # Controller
php artisan make:request Name        # Form Request
php artisan make:test Name --pest    # Pest test
php artisan route:list               # List all routes
php artisan wayfinder:generate       # Regenerate typed route functions
```

## Composer Scripts

```bash
composer run dev                     # Start all dev services (PHP + Vite)
```

## Rules

- Run `vendor/bin/pint --dirty --format agent` after modifying PHP files
- ESLint ignores auto-generated files (`actions/`, `routes/`, `wayfinder/`, `components/ui/`)
- Never use `eslint-disable` comments — fix the underlying issue
- Never edit `resources/js/components/ui/*` — re-add via shadcn CLI
- Never edit `resources/js/actions/` or `resources/js/routes/` — auto-generated by Wayfinder
